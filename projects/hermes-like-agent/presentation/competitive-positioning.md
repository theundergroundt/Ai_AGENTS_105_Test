# Competitive Positioning

## Comparison Frame

- OpenClaw
- Hermes
- NemoClaw
- DeskMate

## OpenClaw

- 정체성: personal AI assistant
- 강점: 멀티채널, gateway, session/state orchestration
- 한계: DeskMate 기준으로는 범위가 너무 넓고 채널 중심이다

## Hermes

- 정체성: self-improving agent platform
- 강점: memory, skills, automation, delegate
- 한계: MVP 관점에서 복잡도가 높고 플랫폼 지향이 강하다

## NemoClaw

- 정체성: secure runtime reference stack
- 강점: sandbox, policy, onboarding, inference routing
- 한계: 제품 기능 차별화보다 운영 hardening에 가깝다

## DeskMate

- 정체성: AI desk assistant
- 강점: dashboard + IoT + work/routine integration
- 전략: 하나의 비서 UX를 유지하면서 기능형 agent를 내부적으로 운영

## Key Slide Message

- 우리는 오픈소스를 조합한 것이 아니라, 세 오픈소스에서 각각 다른 강점을 분석해 `우리 제품 경험`으로 다시 설계했다.

## Reusable One-Liners

- OpenClaw는 "assistant control plane"을 보여줬다.
- Hermes는 "assistant operating philosophy"를 보여줬다.
- NemoClaw는 "secure runtime boundary"를 보여줬다.
- DeskMate는 이 세 가지를 참고하되, `사용자의 책상 위 경험`으로 재정의한다.
