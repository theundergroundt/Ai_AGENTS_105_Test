# Demo Storyline

## Scenario 1: Scheduler

- 사용자가 웹에서 일정 알림을 요청한다.
- DeskMate가 Scheduler Agent를 선택한다.
- 대시보드에 예약 상태가 보인다.
- 시간이 되면 IoT 장치가 LED 또는 부저로 알린다.

## Scenario 2: Wellness

- 사용자가 오늘 상태나 식사 관련 질문을 입력한다.
- DeskMate가 Wellness Agent를 선택한다.
- 결과가 대시보드에 기록된다.
- 완료 상태가 IoT 장치에 반영된다.

## What The Audience Should Notice

- 사용자는 하나의 assistant와 대화한다.
- 내부에서는 요청 종류에 따라 agent가 나뉜다.
- 결과가 채팅에서 끝나지 않고 웹과 IoT로 이어진다.

## Demo Emphasis

- "멀티 에이전트"보다 "assistant가 실제로 개입한다"는 느낌을 강조한다.
- 단순한 답변보다 상태 가시화와 실제 알림 흐름을 보여준다.
