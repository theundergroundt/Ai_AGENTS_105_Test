# 정리 원칙

## 목적

- `gatewayEx` 안의 gateway 관련 코드 이해를 단계적으로 누적한다.
- 나중에 다시 볼 때 "어디서 시작하고 무엇을 읽어야 하는지" 바로 알 수 있게 만든다.

## 메모 작성 규칙

- 사실: 코드에 실제로 있는 구조와 동작
- 해석: 왜 이런 분리가 되었는지에 대한 설명
- 질문: 아직 확인이 필요한 포인트

위 세 가지를 가능하면 구분해서 적는다.

## 파일 추가 규칙

- 구조 분석은 `01-구조/`
- 요청 흐름과 실행 순서는 `02-흐름/`
- 특정 소스 파일 분석은 `03-파일/`

## 현재 해석

- `gatewayEx`는 gateway 전체 구현이 아니라 핵심 흐름을 이해하기 위한 발췌본이다.
- 네 파일이 순서대로 연결되며, 특히 `startGatewayServer -> createGatewayRuntimeState -> createGatewayHttpServer/attachGatewayUpgradeHandler -> attachGatewayWsConnectionHandler` 흐름이 중심이다.
