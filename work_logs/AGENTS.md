# 에이전트 작업 지침

이 지침은 `work_logs`와 그 하위 모든 폴더에 적용된다.

## 목적

- 의미 있는 작업 세션마다 날짜 기준 작업 로그를 남긴다.
- 요청 내용과 작업 결과를 가볍게 추적할 수 있도록 기록한다.

## 폴더 규칙

- 날짜별로 `YYYY-MM-DD` 형식의 폴더를 하나 만든다.
- 해당 날짜의 로그 파일은 그 날짜 폴더 안에만 둔다.
- 다른 날짜의 로그를 같은 폴더에 섞지 않는다.

## 파일 이름 규칙

- 로그 파일은 Markdown 형식으로 만든다.
- 파일명은 24시간제 시각으로 시작한다.
- 권장 형식:
  - `HHMM-작업-요약.md`
  - 예시: `1415-claw3d-초기-분석.md`

## 필수 기록 항목

각 로그 파일에는 아래 항목이 반드시 들어가야 한다.

1. 시간
2. 사용자 요청
3. 작업 요약
4. 변경 사항

## 권장 템플릿

```md
# 작업 로그

## 시간

2026-04-14 14:15

## 사용자 요청

- 요청 내용

## 작업 요약

- 수행한 작업
- 핵심 결과
- 남은 이슈 또는 다음 단계

## 변경 사항
- 이전과 비교해서 변경된 사항 요약
- 변경 이유 요약
- 개선점 요약 
```

## 작성 규칙

- 짧고 사실 중심으로 기록한다.
- 터미널 원문을 길게 붙이지 말고 결과만 요약한다.
- 작업 방향이 중간에 바뀌면 최종 기준으로 요약을 정리한다.
- 막힌 경우에는 무엇이 막혔는지 분명히 적는다.

## 보안 규칙

- 비밀값, 토큰, 개인 인증 정보는 로그에 남기지 않는다.
- 큰 소스코드 덤프를 로그에 그대로 붙이지 않는다.
