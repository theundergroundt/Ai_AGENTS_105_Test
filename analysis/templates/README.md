# Analysis Templates

이 폴더는 `시스템 설계 분석`용 템플릿과 공통 기준 문서만 둔다.

## 포함 문서

- `system-design-analysis-guide.md`
  시스템 설계 분석 과목 기준으로 어떤 질문을 던지고 어떤 순서로 분석할지 정리한 가이드
- `general-project-analysis-template.md`
  일반 소프트웨어 프로젝트 분석을 시작할 때 복사해서 쓰는 템플릿
- `agent-project-analysis-template.md`
  agent 시스템을 기능과 흐름 중심으로 분석할 때 복사해서 쓰는 템플릿

## 원칙

- 템플릿에는 사실 대신 질문과 빈 구조를 둔다.
- 특정 프로젝트의 결론이나 해석은 넣지 않는다.
- 실제 분석 결과는 항상 `../projects/` 아래에 둔다.
- 중복되는 비교표나 체크리스트는 별도 파일로 늘리지 않고, 가이드와 템플릿 안에 흡수한다.
