# System Design Analysis Guide

이 문서는 에이전트 시스템을 `시스템 설계 분석` 과목 관점에서 정리하기 위한 공통 가이드다.

## 1. 분석 목적

- 시스템이 해결하는 문제와 요구사항을 명확히 한다.
- 구조를 설명하는 데서 멈추지 않고 설계 선택의 이유를 정리한다.
- 코드에서 확인한 근거를 바탕으로 우리 프로젝트에 적용 가능한 판단을 남긴다.

## 2. 이 가이드가 맞추는 과목 관점

이 가이드는 아래 항목을 중심으로 구성한다.

- 문제 정의
- 기능 요구사항과 비기능 요구사항
- 시스템 경계와 외부 의존성
- 컴포넌트 분해와 인터페이스
- 데이터 흐름과 제어 흐름
- 품질 속성과 trade-off
- 확장성, 운영성, 보안, 유지보수성
- 구현 근거와 설계 판단 연결

즉, 단순 기능 소개나 코드 나열보다 `왜 이런 구조가 필요했는가`를 설명하는 데 초점을 둔다.

## 3. 먼저 고정할 분석 질문

### 3.1 문제 정의

- 이 시스템은 어떤 사용자 문제를 해결하는가?
- 주요 사용자와 사용 시나리오는 무엇인가?
- 챗봇, 코딩 에이전트, 워크플로 자동화, 멀티 에이전트 중 어디에 가까운가?

### 3.2 요구사항

- 핵심 기능 요구사항은 무엇인가?
- 중요한 비기능 요구사항은 무엇인가?
- 속도, 정확도, 비용, 확장성, 보안, 운영성 중 무엇을 우선하는가?
- 실패 허용 범위와 실패하면 안 되는 지점은 어디인가?

### 3.3 구조

- 시스템 경계는 어디까지인가?
- 주요 컴포넌트는 어떻게 분리되는가?
- 컴포넌트 간 인터페이스와 책임은 무엇인가?
- 외부 모델, 저장소, 도구, UI와는 어떻게 연결되는가?

### 3.4 런타임 흐름

- 입력은 어디서 들어오는가?
- 라우팅, 계획, 도구 호출, 결과 통합은 어떤 순서로 수행되는가?
- 오류, 재시도, 사용자 승인 흐름은 어떻게 처리되는가?

### 3.5 품질 속성과 trade-off

- 이 구조가 잘하는 것은 무엇인가?
- 반대로 감수한 복잡도는 무엇인가?
- 운영과 테스트는 얼마나 쉬운가?

### 3.6 우리 프로젝트 적용성

- 바로 가져올 수 있는 설계는 무엇인가?
- 지금 범위에는 과한 설계는 무엇인가?
- MVP와 확장 단계로 나누면 무엇이 달라지는가?

## 4. 분석 단위

복잡한 에이전트 시스템은 아래 4단위로 나눠 보면 정리가 잘 된다.

### 4.1 제품 관점

- 사용자에게 보이는 핵심 가치
- 주요 사용 시나리오
- 기능 범위

### 4.2 시스템 관점

- 전체 컴포넌트 구성
- 인터페이스 경계
- 외부 의존성
- 데이터와 제어 흐름

### 4.3 에이전트 관점

- planning
- memory
- tool use
- reflection
- human-in-the-loop
- multi-agent coordination

### 4.4 코드 관점

- 폴더 구조
- 엔트리 포인트
- 핵심 모듈
- 확장 포인트
- 결합도와 응집도

## 5. 추천 분석 절차

1. 프로젝트의 문제 정의와 사용자부터 적는다.
2. README와 문서에서 시스템 경계와 외부 의존성을 먼저 잡는다.
3. 엔트리 포인트와 핵심 실행 흐름만 따라간다.
4. 에이전트 고유 설계와 일반 시스템 설계를 구분해서 적는다.
5. 품질 속성과 trade-off를 따로 정리한다.
6. 마지막에 우리 프로젝트 적용성을 적는다.

## 6. 꼭 남겨야 하는 근거

- 공식 문서 링크
- 실제 코드 경로
- 핵심 클래스, 함수, 모듈 이름
- 설정 파일 또는 프롬프트 위치
- 테스트나 운영 관련 근거

## 7. 권장 템플릿 선택

- 일반 소프트웨어 프로젝트:
  `general-project-analysis-template.md`
- agent, coding agent, workflow agent, multi-agent 시스템:
  `agent-project-analysis-template.md`

## 8. 산출물

- 프로젝트별 분석 문서
- 필요 시 비교 메모
- 우리 프로젝트 적용 메모

산출물의 최소 기준은 `문제 정의 -> 요구사항 -> 구조 -> 흐름 -> trade-off -> 우리 시사점`이 모두 남아 있는 것이다.
