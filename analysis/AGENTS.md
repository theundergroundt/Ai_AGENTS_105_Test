# 분석 폴더 작업 지침

이 지침은 `analysis`와 그 하위 폴더에 적용된다.

## 목적

- 요청에 따른 분석 결과를 누적한다.
- 외부 소스별 이해도를 점진적으로 쌓는다.
- 우리 프로젝트에 필요한 해석과 설계 포인트를 분리해서 기록한다.

## 배치 규칙

- 일회성 또는 요청 단위 메모는 `requests`에 둔다.
- Claw3D, OpenClaw, Hermes 같은 소스별 누적 분석은 `upstreams`에 둔다.
- 우리 프로젝트 관련 분석은 `projects`에 둔다.

## 작성 규칙

- 파일명만 보고도 주제가 보이게 쓴다.
- 사실, 해석, TODO를 섞어 쓰지 않는다.
- 사용자 질문으로 설명이 더 쉬워졌으면 관련 문서를 같이 갱신한다.
