# Git Upstream

## 한 줄 정의

`upstream`은 내가 참고하거나 따라가는 원본 저장소 쪽 remote 이름으로 많이 쓰인다.

## 쉬운 설명

내가 어떤 오픈소스를 fork해서 가져오면 보통 두 군데를 보게 된다.

- 내 계정의 fork 저장소
- 원래 공개 저장소

이때 보통:

- 내 fork는 `origin`
- 원래 공개 저장소는 `upstream`

으로 등록한다.

## 왜 쓰나

- 원본 프로젝트가 업데이트되면 그 변화를 따라오기 쉽다.
- 내 변경과 원본 변경을 구분해서 보기 좋다.
- fork 기반 협업에서 기준 저장소가 무엇인지 명확해진다.

## 예시

- `forks/claw3d` 안에서 작업 중
- 내 GitHub fork: `origin`
- 원본 Claw3D 저장소: `upstream`

그러면 원본 최신 변경을 확인할 때 `upstream` 기준으로 본다.
